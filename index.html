Called — End-to-End App Flow Map

A snapshot of every primary screen in the Called web app, captured by walking the product as a logged-in admin. Each card shows the screen, where it lives, and where its buttons take you. Use it for an executive walkthrough or feed flow.json to an LLM to surface UX issues.

How to read this

  • Each phase covers one user surface (auth, shell, lifecycle of a community, etc.) and is independently navigable from the left.
  • Each card is one screen state. URL is the route (Wouter); auth tags whether it's reachable signed-out. Click the screenshot to view full-size.
  • Actions expands the list of clickable affordances on that screen and where each one takes you. Targets prefixed with modal: are overlays; external: are off-site links.
  • Back-button is conventional browser history everywhere we tested — closing a modal returns to the underlying route, no custom back-stack layered on top.
  • For LLM consumption: the same content is in flow.json, plus the ux_observations array there summarizes friction points worth fixing.

  • Top bar — right: What's New badge (opens Headway widget) and Notifications bell (opens dropdown panel).
